Core Capabilities

Core Features

Designed as an end-to-end writing system, not a single prompt wrapper.

⚙️

Multi-Stage AI Generation Pipeline

5-step writing flow: Draft -> Reader Feedback -> Revision -> QA Review -> Final Draft. Each stage routes to fit-for-purpose models instead of one-shot generation.

📚

Knowledge Base System

Structured character bibles, worldbuilding docs, and plot outlines are injected before generation with strict per-project isolation.

🧩

Hard Constraint Engine

Anchor text enforcement, word count gates, pattern blocking, pronoun rules, chapter ending correction, and formatting normalization.

🛰️

Multi-Model Orchestration

Dynamic model selection per stage with token estimation, prompt optimization, and OpenAI-compatible API endpoints via configurable base URLs.

📡

Real-time Streaming Output

Server-Sent Events stream content as it is generated, with completion auto-save and built-in retry handling for transient failures.

🗃️

Global Bible Management

Cross-project character and setting database backed by SQLite, auto-injected into writer and QA prompts for long-form consistency.

Pipeline Walkthrough

How the AI Writing Pipeline Works

Each stage has a focused responsibility and hands over structured outputs to the next stage.

Stage 1

First Draft

Knowledge injection + prompt assembly, then AI generates the initial chapter draft.

->

Stage 2

Reader Feedback

A reader simulator agent evaluates engagement, pacing, and hook effectiveness.

->

Stage 3

Second Draft

Feedback is incorporated to rewrite weak sections and improve narrative flow.

->

Stage 4

QA Review

Automated review over Plot Logic, Writing Style, Commercial Appeal, and Structural Compliance.

->

Stage 5

Final Draft

Hard constraints are enforced, chapter ending is corrected, and output is auto-saved.
